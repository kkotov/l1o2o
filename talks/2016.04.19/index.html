L1T O2O workflow

Bookkeeping relies on L1TriggerKeyList (keyList) listing all payloads ever written to CondDB

L1T O2O gets evoked by the Function Manager and performs two steps:

1) Write payloads step:

  • do nothing if the current TSC key is already in the keyList, else:
    • loop over all records in TSC key and check if (record,key) pair is already in keyList
    • if not, write payload to DB, and extend keyList with the new (record,key,db_token)

2) Write IOVs step (i.e. write to the IOV table of CondDB (db_token,first_run_valid) pairs):

  • update payload's IOV for the TSC key by its db_token
  • loop over all (record,key) pairs in the TSC key
    • extract db_token from the keyList
    • update IOVs for the db_token

Subsystem modules for the L1T O2O framework

Each L1T subsystem instantiates two ESProducers (see examples in L1TriggerConfig):

1) *ObjectKeysOnlineProd* exploring tables in slide #3 and relating record to key

2) *OnlineProd* pulling actual online DB configuration and constructing the payload:

  • from online DB by the key from previous step
  • the later uses XML parser and CondFormats helper classes to construct the payload
