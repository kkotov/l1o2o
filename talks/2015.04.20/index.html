Current setup

Original recipe uses CondDB V1, SLC5, CMSSW_4_2_3, but it still runs (probably not for a long):

l1emulator@l1-o2o.cms > crontab -l
*/10 * * * * /nfshome0/popcondev/L1Job/validate-l1Key.sh
*/1 * * * * /nfshome0/popcondev/L1Job/validate-o2o-wbm.sh
*/15 * * * * /nfshome0/popcondev/L1Job/o2o-dummy.sh


New code uses CondDB V2 and CMSSW_7_2_0; it is ran by the Trigger Function Manager via ssh:

l1emulator@cms-kvm-31.cms > ls o2o-setIOV-l1Key.sh
/nfshome0/l1emulator/run2/o2o/pro/o2o-setIOV-l1Key.sh


In /nfshome0/l1emulator/run2/o2o/v3/ there also exists a version ported to CMSSW_7_4_0

  • O2O access OMDS and save Configs and RSs as records for CMSSW L1T config producers
  • The records are labeled using L1O2OTags so that each tag matches CRAFT09*_hlt pattern
  • Additional L1TriggerKey record holding the exact OMDS keys is also created
