L1T O2O: status and prospects

Khristian Kotov

Status (current version)

The workflow:

  • code is invoked by a Function Manager (FM) call that provides Run # and L1_KEY arguments

  • using L1_KEY it identifies subsystem specific keys and run the online producers

  • online producers navigate further down the tables and pack the info in CondFormats

  • no feedback to FM is provided

Logging and monitoring:

  • system produces a very comprehensive set of log files

  • monitoring is done by looking at the logs on daily basis

Possible failures:

  • development in pro version parallel to the data taking

Prospects (future development addressing upgrade)

The new offline CondFormats are already in 80X

Current version of L1T O2O knows nothing about the upgrade (apart from S1CALOL2)

Swatch team is working on the online DB schema for all of the new systems:

  • configuration is stored as XMLs uniformly across all subsystems

Once online tables are done we will proceed with the online producers
