The L1T O2O development

The detector upgrade renders some payloads obsolete and requires new payloads

We preserve the Run I O2O (legacy O2O) as is thus ensuring format compatibility

Development happens within the new O2O (upgrade O2O) which is a "clone" of the legacy O2O

  • all legacy payloads are still O2O'ed
  • new payloads can be accommodated independently

New code (none of it is yet in git):

  • L1TriggerKeyStage1 and L1TriggerKeyListStage1 payloads
  • CondTools/L1TriggerStage1 package
  • L1TriggerConfig/<Some_Online_Producers>

(we consider to change "Stage1" suffix to something more generic)
