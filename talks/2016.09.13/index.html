Thoughts and plans

  • L1T O2O for validating online XML

    • we've identified TSC/RS keys where config XML had missing values (unintentionally)
    • this could be easily caught if preparing a new key would include "pilot-O2O-running"
    • solution is available and we would like to add it to L1 DOC instructions

  • Reconstructing the history

    • we want to replay the current version of the L1T O2O on the whole 2016
    • the resulting masks and prescales is a "must have" information for many analyses

  • Configurable non-configurables

    • in some cases/systems rarely changed LUTs are not kept track of in the OMDS
    • a new L1T O2O functionality is needed for system experts to be able to update these LUTs manually in the CondDB
